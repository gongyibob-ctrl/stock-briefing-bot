# 全市场板块涨幅 + 板块内优质股数据源评估 — 2026-05-22

## 背景

`brief.py` Layer 1 目前已有龙虎榜 / 大宗 / 融资融券 / 北向资金；现在要追加"今日最强行业板块 + 板块内质地最好的企业"模块。

约束：
- 不走东财行业接口（`ak.stock_board_industry_name_em`），用户已被它的间歇限频坑过
- 不要付费 API、不要爬要登录的站
- 失败不能阻塞主流程

## 关键洞察

1. **东财直接 HTTP 也 502**，不是 akshare 的问题。整个东财 API 群（quote、push2、push3）今晚仍是孤岛。但**同花顺和新浪都能拿到全市场板块数据**，证明替代源充足。
2. **同花顺有反爬，但单日单次请求频率绰绰有余**：实战中一天 1 次 brief.py，**~6 次请求**（1 次板块列表 + 5 次板块详情）不会触发限频；我在今晚测试中密集打了 20+ 次才被限速。这跟用户原来用 akshare 调东财 ths 接口被坑的根因不同——akshare 套了一层不稳的代码、且通过东财通道而非直连同花顺。
3. **新浪是兜底而非主力**：新浪的"板块涨幅"接口**没有**——必须遍历 31 个申万一级节点本地聚合，密集请求会触发 IP 封禁（实测一次封 5 分钟以上）。所以仅作 fallback，不作主源。
4. **板块成分股自然包含质地评分输入**：同花顺详情页一次就返回成分股的"流通市值/PE/换手/量比/成交额"，可以**先做规则粗筛**（去微盘+去亏损）再调用 baostock 补 ROE/净利润。一只板块约 5-8 只候选股、3 个板块 × 8 ≈ 24 个 baostock query ≈ 12-15 秒，整体可控。
5. **板块 LLM 输出价值远高于纯排序**：用户已在用 LLM 写个股简报；板块内"为什么 A 好于 B"的叙述价值更高，规则排序很容易把"PE 最低的"当冠军（但低 PE 可能是周期股顶部），让 LLM 综合多个维度判断更稳。

## 推荐方案：C 混合方案（规则粗筛 + LLM 精选）

```
[同花顺 1 次请求] 全市场行业板块涨幅 Top 5
    │
[同花顺 5 次请求] 每个板块的成分股前 20（按涨幅 desc）
    │
[规则粗筛] 流通市值 ≥ 30 亿 + PE 数字（剔亏损）+ 当日上涨 → Top 6 候选池
    │
[baostock] 给 5 板块 × 6 只 = 30 只候选股补 ROE/净利润/同比
    │
[LLM] 每个板块单独调一次，给候选池 6 只 → 输出 Top 3 + PASS 名单 + 显式理由
    │
[备用源] 同花顺 retry 4 次仍 401 → 自动 fallback 新浪申万一级聚合
```

### 为什么选 C 不选 A/B

- **纯规则 A** 简单稳，但"质地评分"是模糊问题——给 ROE/净利润/PE 加权累加得到的"分数"在边界 case 上不直观。规则适合粗筛、不适合最终排序。
- **纯 LLM B** 把 20 只全部丢给 LLM 让它过滤，浪费 token 且 LLM 在"剔除明显垃圾股"这种确定性任务上没有优势。
- **混合 C** 用规则把"流通市值 < 30 亿、亏损"这种硬性垃圾过掉，把"质地最优 3 只"这种需要权衡的判断交给 LLM，**LLM 还会输出叙述**（"沪电股份估值最低 PE 44 同时 ROE 28%，性价比突出"），直接进飞书简报无需二次加工。

## 限制与备案

1. **同花顺反爬随机性**：密集测试时（一小时内 10+ 次）会进入限频，单次请求都返回 401。我在 `_ths_get_with_retry` 加了指数退避 4 次重试（2.5s → 20s），全失败后 fallback 到新浪。实战一天一次不会触发，但**多次手动调试时要拉开间隔**。
2. **新浪 fallback 的板块涨幅是本地市值加权计算，不是官方板块指数涨幅**：申万一级 31 个板块挨个拉成分股本地加权——新股 N/C 开头、ST、停牌、当日 |涨幅| > 25% 已过滤，但与官方申万指数仍可能有 0.1-0.3% 偏差，结论用作"今天涨得最多的板块"足够。
3. **新浪 fallback 耗时 ~30-70 秒**：因为要遍历 31 个节点。同花顺主源耗时 ~5 秒。
4. **同花顺概念板块不可用**：仅行业板块。如果未来想加"今日热门概念"（人形机器人 / AIGC 等），需要额外找源——网易概念 / 雪球（要 cookie，ToS 风险）/ 自建概念-成分股映射，**当前模块不覆盖**。
5. **基本面是最近年报为主**：2025 年报多数公司已在 2026 年 3-4 月披露，5 月起覆盖 95%+；少数未披露的回退最近季报，再回退则该字段缺失（LLM prompt 会显示"无数据"，不影响其它候选）。
6. **LLM 偶发返回空**：实测 deepseek-chat 3 个板块的 LLM 调用偶有 1 个返回空（API 偏稳但非 100%）。`format_sector_section` 会显示 "⚠️ LLM 返回空" 并降级显示候选池数据，用户能立刻发现并自己读数据。
7. **板块详情默认只取涨幅 Top 20**：意味着板块内"大市值但今天没涨"的龙头可能不在候选池里。这是 trade-off：用户要的是"强势板块里的优质股"，今天没涨的股票通常已经在 Top 20 之外，候选池里追求"已经在涨的优质龙头"更符合 brief 场景。如要扩展，可在 `fetch_ths_sector_cons` 接受 `page` 参数翻页，但会增加触发限频的概率。
8. **不做"概念板块" / "热点题材"**：同花顺 `gn/` 强反爬，绕过 ToS 风险高。如果要做，建议手动维护"我关心的题材-成分股"映射 + baostock 直接拉成分股的日线涨幅做本地聚合，比对接热点 API 更可控。

## 实施

代码见同目录 `sector_strength_patch.py`。新增依赖：

```bash
pip install curl_cffi
# baostock 已在前一个 patch（quote_source_patch.py）引入
```

`brief.py` 集成方式（在 `fetch_market_tables()` 之后，`render_markdown` 渲染 Layer 1 之前）：

```python
from docs.sector_strength_patch import run_sector_module
# main() 里:
sectors, sector_md = run_sector_module(top_n_sectors=5, pool_size=6, call_llm=True)
# 把 sector_md 拼到 render_markdown 输出的 "## 🌐 全市场态势" 那一段末尾即可
```

返回 schema 在 `sector_strength_patch.py` 各函数 docstring 中已写明。
